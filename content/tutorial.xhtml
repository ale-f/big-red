Getting started

This guide will take you through the construction of a simple bigraph using Big Red. (Remember that you can move this guide around; drag its tab to the bottom of the window to see the main workspace.)

  1. Add ports to the bigraph

    Bigraphs are useless without connections joining nodes together, and nodes can only be joined at ports - designated connection points. Let's add a few ports to get started.

    Go to the Bigraph menu, then select Ports.... The Ports wizard will appear.

    The option we want - Add a new port - is already selected, so click Next. The port editor will appear.

    We're going to model a simple office environment, so let's create the first port: telephone-cable, which models the wire that connects a telephone to the telephone system. Type this into the Name box.

    telephone-cable can connect to the telephone system, so we'll model that as the port telephone-system. Click Add; a new entry appears in the large frame, ready for editing. Type telephone-system into the new entry. (If you make a mistake, you can click it again to edit it.)

    That's it; click Finish to create this port. (Notice that you don't need to make a separate telephone-system port - it's created implicitly.)

  2. Adding a control

    Bigraphs have the concept of controls (essentially classes which govern some aspects of a node's display and function); each node on the bigraph has a control, which dictates its appearance and its ports. Since we're modelling an office environment, we'll need two controls to start: Building and Telephone.

    Go to the Bigraph menu, then select Controls.... The Controls wizard will appear.

    As before, Add a new control has already been selected, so click Next.

    The control editor might look complicated, but don't worry! It's actually quite easy to use. Type Building in the Name field, and then - using the drop-down boxes - set the Top port to telephone-system.

    The port is now configured; any Telephones we add to the bigraph will be able to connect to Buildings.

    Let's change the look of this control. The default size of 50x50 is fine, but it should be resizable, so tick the Allow resizing box at the bottom - and change the Appearance drop-down box to Oval.

    Once you've done that, we can move on to actually creating the bigraph.

  3. Adding the nodes

    You should be back at the main window at this point. From the list of tools (the palette), choose Root.

    The formal language of bigraphs only allows us to create nodes if they're inside a root, so we need one of those before we can go any further. Move the mouse into the blank white area (the canvas), and draw a large rectangle with it (click, hold, and drag). This will add a root - shown as a white rectangle with rounded corners and a dashed border - to the bigraph.

    (Notice the black border around it, and the square handles around the border; this tells us that this item is selected. When something is selected, you can click and drag the handles to resize it - or you can click and drag the figure itself to move it around.)

    Now select the Node tool from the palette, and draw another large rectangle - inside the root.

    This node doesn't have a control, so it has no ports and a very dull appearance. Double-click it, and the Properties view will appear on the left-hand side of the window.

    Its control is Unknown. This is no good; obviously we want it to be Building. Click on the Unknown text, and it'll become another drop-down box - which you can use to choose Building!

    (Notice the red dot at the top of the Building figure; this is how Big Red shows ports.)

  4. Adding the edges

    Adding connections between nodes - edges - is easy. When you use the Edge tool to connect two nodes, Big Red will automatically work out whether or not they have compatible ports - and it'll link those ports together for you, too.

    Select the Edge tool, and use it to draw a line between any of the Telephones and the Building. The result will be something like this:

    Big Red uses Manhattan routing for edges, so - rather than just using a straight line between the source and the destination - the line can only turn ninety degrees at a time. This may not seem very useful now, but when you have lots of edges on a diagram, it makes it much more readable!

    Connect the remaining two Telephones to the Building.

    Congratulations - you've drawn your first Big Red bigraph!

  5. Saving and loading

    To save your masterpiece, go to the File menu, and choose Save. The standard file dialog will appear; call your bigraph Office, then click OK.

    If you don't add one yourself, Big Red will automatically append a .bigred suffix to the filename - so the file actually ends up being called Office.bigred.

    Opening a previously-saved bigraph is similarly intuitive; go to File and choose Open.

That's all for this guide; now you know how to add ports and controls to a bigraph, how to actually draw one, and how to save it out as a file.

Of course, Big Red can do lots more - but the best way to learn is through experimentation! For example, why not try using the Properties view to change the colour of one of the nodes in your diagram?
